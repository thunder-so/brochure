Deploy full-stack server-side rendered applications on AWS using Lambda for SSR, S3 for static assets, and CloudFront as the global CDN.

Thunder provides the [@thunder-so/thunder](https://github.com/thunder-so/thunder) package with framework-specific constructs (`Nuxt`, `Astro`, `SvelteKit`, `TanStack Start`, `SolidStart`, `AnalogJS`) as well as a generic `Serverless` construct for any Nitro or Vite-based framework.

## Supported Frameworks

- [Nuxt](https://nuxt.com/)
- [Astro](https://astro.build/)
- [TanStack Start](https://tanstack.com/start/latest)
- [SvelteKit](https://kit.svelte.dev/)
- [Solid Start](https://start.solidjs.com/)
- [AnalogJS](https://analogjs.org/)
- Any Nitro or Vite-based SSR framework via the generic `Serverless` construct

**CloudFront** routes requests at the edge — static assets are served directly from **S3** while dynamic SSR requests are forwarded to **Lambda**, which renders pages on-demand with pay-per-request pricing and automatic scaling.

## Quick Start

### Installation

```bash
bun add -D tsx @thunder-so/thunder
```

### Basic Setup

```bash
mkdir stack && touch stack/index.ts
```

### Configuration

Use the framework-specific construct for best defaults, or `Serverless` for any other framework:

```ts
// stack/index.ts
import { Cdk, Nuxt, type NuxtProps } from '@thunder-so/thunder';

const config: NuxtProps = {
  env: {
    account: 'your-account-id',
    region: 'us-east-1',
  },
  application: 'my-app',
  service: 'web',
  environment: 'production',
  rootDir: '',
  serverProps: {
    runtime: Cdk.aws_lambda.Runtime.NODEJS_22_X,
    architecture: Cdk.aws_lambda.Architecture.ARM_64,
    memorySize: 1792,
    timeout: 10,
    keepWarm: true,
  },
};

new Nuxt(
  new Cdk.App(),
  `${config.application}-${config.service}-${config.environment}-stack`,
  config
);
```

### Deploy

```bash
bunx cdk deploy --all --app="bunx tsx stack/index.ts"
```

## Advanced Configuration

### Server Runtime

Fine-tune Lambda performance with memory, timeout, and warm-up settings:

```ts
const config: NuxtProps = {
  // ... other props
  serverProps: {
    runtime: Cdk.aws_lambda.Runtime.NODEJS_22_X,
    architecture: Cdk.aws_lambda.Architecture.ARM_64,
    memorySize: 1792,
    timeout: 10,
    keepWarm: true,
    streaming: true, // enable response streaming (Nuxt/Nitro)
    variables: [
      { NODE_ENV: 'production' },
    ],
    secrets: [
      { key: 'DATABASE_URL', resource: 'arn:aws:secretsmanager:us-east-1:123456789012:secret:db-url-abc123' },
    ],
  },
};
```

### Configuration

```ts
const config: NuxtProps = {
  // ... other props
  accessTokenSecretArn: 'arn:aws:secretsmanager:us-east-1:123456789012:secret:github-token-XXXXXX',
  sourceProps: {
    owner: 'your-username',
    repo: 'your-repo',
    branchOrRef: 'main',
  },
  buildProps: {
    runtime: 'nodejs',
    runtime_version: '22',
    installcmd: 'bun install',
    buildcmd: 'bun run build',
  },
};
```
